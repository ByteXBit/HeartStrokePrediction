Introduction

This academic project aims to develop a stroke prediction system using health parameters through Machine Learning (ML) and Deep Learning (DL) techniques. It involves the full ML pipeline‚Äîdata loading, preprocessing, visualization, handling imbalance, scaling, modeling, and evaluation. The ultimate goal is to build a robust and interpretable system that can aid in healthcare diagnostics.

üìÅ Dataset Description

The dataset used is from Kaggle, titled "Healthcare Dataset Stroke Prediction". It contains medical information of 5,110 patients and whether they had a stroke.

Features include:

  • id - Unique identifier (removed during preprocessing)
  • gender - Male, Female, or Other
  • age - Patient's age
  • hypertension - 0: No, 1: Yes
  • heart_disease - 0: No, 1: Yes
  • ever_married - Yes or No
  • work_type - Type of employment
  • Residence_type - Urban or Rural
  • avg_glucose_level - Average glucose level
  • bmi - Body Mass Index
  • smoking_status - Smoking history
  • stroke - 0: No stroke, 1: Stroke (target)

1. Data Preprocessing

Using libraries like Pandas, Numpy, Matplotlib, and Plotly, we began preprocessing the dataset:

data = pd.read_csv("/content/healthcare-dataset-stroke-data.csv")
data.describe()
data.shape

We found that the bmi column had missing values. Rather than dropping these, we filled them with mean BMI values grouped by gender, marital status, and age group to retain logical integrity.

We dropped the id column since it does not contribute to prediction. A new column age_group was added for better categorization and grouped analysis.

data2['bmi'] = data2['bmi'].fillna(data2.groupby(["gender", "ever_married", "age_group"])["bmi"].transform("mean"))

We also removed rows with gender marked as "Other" as they were statistically insignificant (only one entry).

2. Data Visualization

We used Seaborn, Matplotlib, and Plotly Express for graphical analysis. The following relationships were visualized:

  • Gender vs Stroke
  • Age vs Stroke
  • Hypertension and Heart Disease vs Stroke
  • Marital Status, Work Type, Residence vs Stroke
  • Glucose, BMI, Smoking Status vs Stroke

These helped us identify which variables most strongly impact stroke probability.

3. Handling Class Imbalance

The dataset was imbalanced, with a minority class for stroke cases. We handled this using:

  • Upsampling: Duplicating stroke cases.
  • SMOTE: Synthetic Minority Over-sampling Technique (not shown in this code).

This helped improve model generalization on the minority class.

4. Feature Scaling

To ensure consistency and avoid model bias towards larger values, we applied MinMax Scaling on numerical features like:

  • Age
  • Average Glucose Level
  • BMI

5. Model Building & Training

We trained 4 ML models on the preprocessed dataset:

  • Extra Trees Classifier
  • Random Forest Classifier
  • XGBoost Classifier
  • Gradient Boosting Classifier

Each model was trained using a train-test split (80-20) and evaluated using:

  • Training and Testing Accuracy
  • Classification Reports
  • Confusion Matrices

6. Evaluation & Metrics

Metrics used to evaluate model performance:

  • Accuracy: Correct predictions out of total
  • Precision: True Positives out of Predicted Positives
  • Recall: True Positives out of Actual Positives
  • F1-score: Harmonic mean of precision and recall

Confusion matrices provided visual insights into each model‚Äôs prediction capabilities.
